---
title: Credentials and Secret Scopes
description: Understand the difference between room secrets, project secrets, and image pull secrets.
---

MeshAgent uses different secret types for different scopes. The right question is not just "where do I store a secret?" but "which runtime needs it, and at what scope?"

## Secret types

- **Room secrets** are for room-scoped or identity-scoped credentials used inside a room.
- **Project secrets** are for shared runtime credentials used by deployed services.
- **Image pull secrets** are for pulling private container images before startup.

## Choose by scope

- If the credential belongs to a room or a participant flow, use [Room Secrets](../secrets/room_secrets).
- If the running service needs a shared credential at runtime, use [Project Secrets](../secrets/project_secrets).
- If MeshAgent only needs credentials to pull an image, use [Image Pull Secrets](../secrets/image_pull_secrets).

OAuth-related secret flows for room identities are covered in [OAuth with Room Secrets](../secrets/oauth_flows).
