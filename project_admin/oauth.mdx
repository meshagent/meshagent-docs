title: OAuth Clients
description: Create project-owned OAuth clients for your app and connect them to participant-token issuance.
---

OAuth clients are the project-level auth configuration for your own application.

Use them when you want users to sign in to your app through MeshAgent, then join rooms with the right participant tokens and room grants.

Do not use OAuth clients for backend automation or CI. Use [API Keys](./api_keys) for that.

You do not need an OAuth client for MeshAgent Studio, Powerboards, or normal CLI sign-in. Those flows use MeshAgent's built-in auth.

## How OAuth clients fit

The normal flow looks like this:

1. Create an OAuth client for the project.
2. Send the user through that OAuth flow from your app.
3. After sign-in, have your backend mint [participant tokens](../rest_api/participant_tokens) for the rooms the user should access.
4. Connect the client to the room with that token.

The OAuth client handles user sign-in. The participant token handles room access.

## Where to manage OAuth clients

- Use [MeshAgent Studio](../interfaces/meshagent_studio) for the main UI flow.
- Use the [REST API](../rest_api/overview) or SDKs when you need to provision clients programmatically.

In the REST API, OAuth clients live under the project:

- `POST /accounts/projects/{project_id}/oauth/clients`
- `GET /accounts/projects/{project_id}/oauth/clients`
- `PUT /accounts/projects/{project_id}/oauth/clients/{client_id}`
- `DELETE /accounts/projects/{project_id}/oauth/clients/{client_id}`

## What to configure

When you create an OAuth client, define:

- redirect URIs for your app
- the grant types and response types your app uses
- the scopes your app should request
- any client secret handling required by your flow

Keep the redirect URIs and scopes tight to the app you are actually shipping.

## What this page does not cover

- For CLI sign-in, use `meshagent setup` or `meshagent auth login`.
- For MeshAgent Studio or Powerboards sign-in, use the built-in MeshAgent sign-in flow.
- For backend automation, use [API Keys](./api_keys).
- For room-secret OAuth flows between participants inside a room, use [OAuth With Room Secrets](../secrets/oauth_flows).
