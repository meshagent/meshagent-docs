`meshagent setup` configures Codex and Claude tools to route model requests through MeshAgent.

Use this guide when you want Codex CLI, the Codex desktop app, Claude Code, or the Claude desktop app to use a MeshAgent project instead of talking directly to OpenAI or Anthropic. MeshAgent keeps provider access, project permissions, usage tracking, billing, and budget controls centralized.

MeshAgent provides access to OpenAI and Anthropic automatically. If you want the project to use your own provider credentials instead, configure them in [MeshAgent Studio](https://studio.meshagent.com) under the [Integrations](../../project_admin/integrations) tab.

## Configure Codex and Claude

Start with:

```bash
meshagent setup
```

`meshagent setup` walks you through signing in, choosing a project, and configuring supported Codex and Claude tools installed on your machine.

- sign you in, or let you continue with the current account or switch accounts
- choose or create a project and activate it for the CLI
- create and activate an API key for that project if you do not already have one
- check whether your account can use the LLM proxy for that project
- if Codex is installed, create, update, reuse, remove, or default a MeshAgent Codex profile
- if Claude is installed, configure, update, or remove the MeshAgent Claude settings

You need a project you can use and permission to use the LLM proxy for that project. If your account does not have that access, setup tells you and stops at the normal project activation flow.

## Codex

`meshagent setup` can add, update, remove, or make default a MeshAgent Codex profile for the active project.

### Use Codex in the CLI

If setup made a MeshAgent profile the default, you can usually start Codex normally.

If you kept the profile but did not make it the default, start Codex with that profile:

```bash
codex -p meshagent
```

If you want a one-off MeshAgent Codex session without changing your normal default profile, use:

```bash
meshagent launch codex
```

### Use Codex in the desktop app

If your Codex install supports `--app`, launch the desktop app through MeshAgent with:

```bash
meshagent launch codex --app
```

`meshagent launch codex` is the one-off CLI path. `meshagent launch codex --app` forwards `--app` to Codex so you can use the same MeshAgent project setup in the desktop app when that flag is available in your Codex install.

## Claude

`meshagent setup` can configure Claude to use MeshAgent for the active project, update that configuration later, or remove it when you want to go back to your normal provider account.

### Use Claude in the CLI

If setup made MeshAgent your normal Claude path on that machine, start Claude normally.

If you want a one-off MeshAgent Claude session without changing your normal Claude setup, use:

```bash
meshagent launch claude
```

### Use Claude in the desktop app

If your Claude install supports `--app`, launch the desktop app through MeshAgent with:

```bash
meshagent launch claude --app
```

`meshagent launch claude` is the one-off CLI path. `meshagent launch claude --app` forwards `--app` to Claude so you can use the same MeshAgent setup in the desktop app when that flag is available in your Claude install.

## Switch back to your regular OpenAI or Anthropic account

If you only used `meshagent launch codex` or `meshagent launch claude`, there is nothing to undo. Those commands only apply MeshAgent for that launch.

If you changed the default local configuration, switch back like this:

- for Codex, use your normal Codex profile with `codex -p <your-profile>`, or rerun `meshagent setup` and either stop using the MeshAgent profile as default or remove it entirely
- for Claude, rerun `meshagent setup` and choose **Remove Claude MeshAgent configuration**

`meshagent launch codex` and `meshagent launch claude` are the easiest way to switch between your normal provider account setup and a MeshAgent project without making MeshAgent the default on that machine.

If you install Codex or Claude after running setup, rerun `meshagent setup` so MeshAgent can detect and configure them.
