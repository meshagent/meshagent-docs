Queues and scheduled tasks are the main building blocks for background and recurring work in MeshAgent.

## The mental model

- **Queues** are how work is delivered asynchronously.
- **Scheduled tasks** are how work is enqueued on a schedule.
- **Agents or services** consume that queued work and perform the job.

This is useful for recurring workflows such as daily digests, periodic reports, imports, monitoring jobs, or automated follow-up work.

## How this fits with agents

Process-backed agents can listen on queue channels and handle queued work without needing a separate runtime model. That means the same agent can answer in chat, respond to mail, and process scheduled queue work.

## Where the mechanics live

- Use the [Queues API](../room_api/queue) for room-level queue operations.
- Use [Process Agent Service](../services_room_containers/packaging_and_deploying/process_agent_service) for a deployment pattern that includes scheduled work.
- Use the [REST API](../rest_api/overview) or [CLI reference](../reference/meshagent_cli_help) when you need to create or manage scheduled tasks programmatically.
