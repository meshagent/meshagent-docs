This workflow is now covered in [Packaging and Deploying Services](./packaging).

Use `meshagent process deploy` when CLI flags already describe the process-backed agent you want. Use `meshagent process spec` when you want MeshAgent to generate a manifest first and you plan to review or customize it before deployment.

## Most common commands

```bash
# Fastest path for a room-scoped process-backed agent
meshagent process deploy --room myroom ...

# Generate a manifest first, then deploy it into one room
meshagent process spec ... > meshagent.yaml
meshagent service create --file meshagent.yaml --room myroom

# Deploy the same manifest project-wide
meshagent service create --file meshagent.yaml --global
```
