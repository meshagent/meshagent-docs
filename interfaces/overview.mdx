MeshAgent provides two primary interfaces for humans:

- [MeshAgent Studio](./meshagent_studio) for developers and operators
  Open the app: [studio.meshagent.com](https://studio.meshagent.com)
- [Powerboards](./powerboards) for end users working with agents in rooms
  Open the app: [app.powerboards.com](https://app.powerboards.com)

Both are included with a MeshAgent account, and both connect to the same underlying projects and rooms.

That means a room is not "in Studio" or "in Powerboards." It is the same room either way. You might build and debug it in Studio, then open that same room in Powerboards for a cleaner end-user experience.

## When to use each interface

**MeshAgent Studio**
- Create and organize projects and rooms.
- Build, test, and debug agents, tools, and services.
- Inspect room sessions with logs, traces, and metrics.
- Manage members, secrets, webhooks, routes, mailboxes, and other project controls.
- App: [studio.meshagent.com](https://studio.meshagent.com)
- Docs: [MeshAgent Studio](./meshagent_studio)

**Powerboards**
- Start quickly with built-in agents.
- Install shared agents and services into rooms.
- Give non-technical users a focused UI for chat, voice, documents, and files.
- Depending on room grants, open the developer console and inspect logs, traces, and metrics.
- Use deployed room experiences without exposing the full project-administration surface by default.
- App: [app.powerboards.com](https://app.powerboards.com)
- Docs: [Powerboards](./powerboards)

## How they work together

The usual split looks like this:

1. Create a project and room in Studio, Powerboards, or the CLI.
2. Use Studio and the CLI to build, test, deploy, and inspect what runs in that room.
3. Use Powerboards when you want people to install or use agents in a simpler end-user interface.
4. Use Studio when you need the fuller developer/operator surface, especially for project administration and broader project management.

This is also why the same room can be useful to different people in different interfaces:

- a developer can inspect logs and traces in Studio
- a room member can talk to the agent and work with room files in Powerboards
- both are working against the same room state, services, documents, and data
