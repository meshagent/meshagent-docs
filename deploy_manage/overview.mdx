This section covers how code becomes available in rooms and how you manage it once it is running.

Use Deploy & Manage when you need to:

- package code as a MeshAgent service
- decide between project services and room services
- deploy process-backed agents, external services, or templates
- manage runtime credentials and image pull credentials
- inspect observability data for running workloads

## What this section does not cover

- Go to [Rooms](../room_api/overview) for room-scoped runtime APIs such as messaging, queues, storage, containers, sync, and room secrets.
- Go to [Agents & Tools](../agents/overview) for process agents, tools, skills, and scheduled work patterns.
- Go to [Project Admin](../project_admin/overview) for project members, billing, domains, mailboxes, and other account-level controls.
