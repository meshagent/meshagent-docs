---
title: Project vs Room Services
description: Choose whether a service should be shared across a project or scoped to one room.
---

One of the first deployment decisions in MeshAgent is whether a service should be available everywhere in a project or only inside one room.

## Project services

Use a **project service** when the capability should be available in every room in the project.

Good fits:

- shared utility agents
- common integrations
- always-on capabilities used across many rooms

## Room services

Use a **room service** when the capability should be scoped to one room or customized differently per room.

Good fits:

- room-specific research agents
- services with room-local configuration
- user-selectable functionality that should not appear everywhere
