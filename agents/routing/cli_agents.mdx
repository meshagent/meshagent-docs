Terminal-based coding agents such as [OpenAI Codex CLI](https://github.com/openai/codex) and [Claude Code](https://claude.ai/code) use LLMs to reason about code, generate changes, and complete development tasks from the terminal. By default, these tools send model requests directly to a provider such as OpenAI or Anthropic.

This guide shows how to run those tools through MeshAgent with `meshagent llm proxy`. That command exposes local OpenAI-compatible and Anthropic-compatible endpoints on your machine, forwards requests through MeshAgent, and prints the values your coding agent should use.

## Why use the LLM Proxy

- Let someone use Codex CLI or Claude Code through MeshAgent even if they do not already have their own provider account configured for that CLI.
- Centralize provider access in MeshAgent instead of configuring raw provider credentials on each developer machine or CI job.
- Route Codex CLI, Claude Code, and similar tools through the same local setup pattern.
- Keep MeshAgent authentication and usage reporting inside the proxy layer while the coding agent continues to work like a normal OpenAI- or Anthropic-compatible client.

## How it works

`meshagent llm proxy` starts a local server, then prints the environment variables your coding agent should use.

By default that local proxy listens on:

- OpenAI-compatible clients such as Codex CLI: `http://127.0.0.1:8766/openai/v1`
- Anthropic-compatible clients such as Claude Code: `http://127.0.0.1:8766/anthropic`

The coding agent talks to that local proxy. The proxy then authenticates upstream with MeshAgent, forwards the request, and reports usage through MeshAgent. The coding agent itself is unchanged.

## Setup: Shared first steps

These steps apply regardless of which CLI agent you are using.

### 1. Connect the CLI to MeshAgent

If you have not set up MeshAgent yet, start with:

```bash bash
meshagent setup
```

This signs you in, connects the CLI to MeshAgent, and selects the active project.

### 2. Start the local proxy

Run:

```bash bash
meshagent llm proxy
```

The command prints a setup block like:

```bash bash
export OPENAI_BASE_URL=http://127.0.0.1:8766/openai/v1
export OPENAI_API_KEY=...
export ANTHROPIC_BASE_URL=http://127.0.0.1:8766/anthropic
export ANTHROPIC_API_KEY=...
```

<Tip>
Use the exact values printed by your running `meshagent llm proxy` session. If you changed the host, port, or bearer token behavior, those values may differ from the defaults shown here. You will use those printed values in the Codex CLI and Claude Code setup below.
</Tip>

### 3. Open a new terminal window and keep the proxy running

Open a new terminal window for Codex CLI or Claude Code, and leave the original `meshagent llm proxy` window running while either tool is using it.

That proxy process is also where you can see usage across Codex and Claude Code sessions while it stays open.

## Codex CLI

[Codex CLI](https://developers.openai.com/codex/cli) supports custom model providers in `~/.codex/config.toml`, including custom base URLs and API key env vars which can be used with the MeshAgent LLM Proxy.

### 1. Install Codex

<CodeGroup>
```bash macOS
brew install openai-codex
```
```bash npm
npm install -g @openai/codex
```
</CodeGroup>

### 2. Configure `~/.codex/config.toml`

Open the config file:

```bash
nano ~/.codex/config.toml
```

Then add a MeshAgent provider and profile:

```toml ~/.codex/config.toml
[model_providers.meshagent]
name = "MeshAgent"
base_url = "http://127.0.0.1:8766/openai/v1"
http_headers = {"Authorization"="Bearer OPENAI_API_KEY_FROM_PROXY"}

[profiles.meshagent]
model_provider = "meshagent"
model = "gpt-5.4"
```

| Key | Description |
|-----|-------------|
| `base_url` | The local OpenAI-compatible endpoint exposed by `meshagent llm proxy`. |
| `http_headers` | Send the exact `OPENAI_API_KEY` value printed by `meshagent llm proxy` as the local proxy bearer token. |
| `model` | Any model available through the OpenAI API. |

<Tip>
Use the exact `OPENAI_API_KEY` value printed by the running `meshagent llm proxy` window. That terminal must stay open while Codex is using the proxy.
</Tip>

### 3. Run Codex with the MeshAgent profile

In the new terminal window, run:

```bash bash
meshagent setup
codex --profile meshagent
```

<Note>
If you previously logged into Codex with a different account, you may need to start `codex`, run `/logout`, exit, then restart with `codex --profile meshagent`.
</Note>

### 4. Verify Codex is using MeshAgent

After Codex starts, run:

```text
/status
```

You should see Codex using the `meshagent` profile you configured in `~/.codex/config.toml`, including output like:

```text
Model provider:       MeshAgent - http://127.0.0.1:8766/openai/v1
```

### 5. Check usage in the proxy terminal

Look at the terminal window where `meshagent llm proxy` is still running. That window shows request activity and usage totals while Codex is using the proxy.

<Note>
Keep that `meshagent llm proxy` window open. Codex depends on that local proxy process and the printed proxy values while this setup is in use.
</Note>

## Claude Code

[Claude Code](https://code.claude.com/docs/en/overview) speaks Anthropic-compatible APIs, so it should use the local `ANTHROPIC_BASE_URL` and `ANTHROPIC_API_KEY` printed by `meshagent llm proxy`.

### 1. Install Claude Code

<CodeGroup>
```bash macOS, Linux, WSL
curl -fsSL https://claude.ai/install.sh | bash
```
```bash Homebrew
brew install --cask claude-code
```
```bash WinGet
winget install Anthropic.ClaudeCode
```
</CodeGroup>

### 2. Export the proxy values into this terminal session

In the new terminal window from the shared setup above, copy the export lines printed by `meshagent llm proxy` into the current shell session:

```bash bash
export ANTHROPIC_BASE_URL=http://127.0.0.1:8766/anthropic
export ANTHROPIC_API_KEY=...
```

<Tip>
Use the exact values printed by the running `meshagent llm proxy` window.
</Tip>

### 3. Start Claude Code

```bash bash
meshagent setup
claude
```

<Note>
If you previously logged Claude Code into a different account, run `claude /logout`, then start `claude` again so it uses the environment variables you set in this terminal.
</Note>

### 4. Verify Claude Code is using MeshAgent

After Claude Code starts, run:

```text
/status
```

You should see Claude Code using the environment variables you configured, including output like:

```text
API Key:             ANTHROPIC_API_KEY
Anthropic base URL:  http://127.0.0.1:8766/anthropic
```

### 5. Check usage in the proxy terminal

Look at the terminal window where `meshagent llm proxy` is still running. That window shows request activity and usage totals while Claude Code is using the proxy.

<Note>
Keep that `meshagent llm proxy` window open. Claude Code depends on that local proxy process and the printed proxy values while this setup is in use.
</Note>
