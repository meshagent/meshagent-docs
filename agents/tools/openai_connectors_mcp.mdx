OpenAI Connectors and MCP servers let a MeshAgent room expose external tools.

In MeshAgent, these tools are typically exposed through the same MCP tool flow:

- **[OpenAI Connectors](https://platform.openai.com/docs/guides/tools-connectors-mcp)** wrap third-party services such as Gmail, Google Drive, Outlook, Microsoft Teams, and Dropbox.
- **[MCP servers](https://modelcontextprotocol.io/introduction)** expose tools over the Model Context Protocol, whether the server is public, self-hosted, or provided by a third party.

For a concrete vendor-specific walkthrough, see [Supabase MCP Guide](./supabase_mcp).

## Fastest path: install from Powerboards

If you are adding a supported MCP integration to a room, the easiest path is often Powerboards.

1. Open the Powerboards installer.
2. Paste the MCP server URL.
3. Let Powerboards inspect the server and guide the install flow.
4. Choose the room where the tool should be installed.

For some servers, this means you can install the tool without writing YAML or deploying a custom service yourself.

Powerboards can also install from a hosted `ServiceTemplate` link when you want to distribute a more opinionated setup flow. For more on those install paths, see [Powerboards](../../interfaces/powerboards).

## CLI path: create the service from an MCP URL

If you want to stay in the CLI, you can create the external MCP service directly from the server URL.

If you want to inspect the generated spec first:

```bash
meshagent service spec \
  --mcp https://mcp.deepwiki.com/mcp
```

If you want to create the service immediately:

```bash
meshagent service create \
  --mcp https://mcp.deepwiki.com/mcp \
  --room quickstart
```

The CLI auto-discovers the MCP server metadata. For servers that expose OAuth registration metadata, the generated service is configured for the dynamic client-registration flow.

## Start an agent that can use MCP tools

Once the MCP service is installed in the room, start an agent that can use it:

```bash
meshagent process join \
  --room quickstart \
  --agent-name agent \
  --channel chat \
  --mcp
```

Open [MeshAgent Studio](https://studio.meshagent.com), join the room, and enable the MCP tool when you want the agent to use it.

If the agent should always have access to the room's MCP tools, use `--require-mcp` instead.

For the deeper difference between dynamic and always-on tool access, see [How Tools and Toolkits Work](./tools_and_toolkits).

## When to write the YAML yourself

Use a hand-written service spec when you need to control the service shape directly, pin specific metadata, or define an OpenAI Connector explicitly.

## Security and setup notes

When adding external tools like MCP servers or OpenAI Connectors, think about:

- what data the external service will receive
- who operates the MCP server or connector-backed system
- whether users should opt in per turn with `--mcp` or whether the tool should always be attached with `--require-mcp`
- whether the server needs additional secrets or OAuth setup
