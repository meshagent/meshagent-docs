Project administration is where you control access, credentials, billing, and routing for your MeshAgent project. Most teams use [MeshAgent Studio](../interfaces/meshagent_studio) for day-to-day changes, the [MeshAgent CLI](../cli/cli_quickstart) for quick workflows, and the [REST API](../rest_api/overview) for automation.

## What you manage

- **Members and roles**: Invite teammates and define what they can do.
- **API keys**: Issue and rotate project-scoped credentials for services and automation (see [API Keys](./api_keys)).
- **Authentication**: Configure OAuth clients and participant access patterns (see [Authentication](./authentication)).
- **Billing and usage**: Track spend, credits, and usage trends (see [Billing](./billing)).
- **Domains**: Map domains to room services and published endpoints (see [Domains](./domains)).

## Where to manage

- **MeshAgent Studio**: The primary UI for projects, members, keys, and billing.
- **MeshAgent CLI**: Quick project changes and scripts (for example API keys and routes).
- **REST API**: Programmatic access for provisioning, CI, or multi-project automation.

## Recommended setup

1. Create a project and invite your team.
2. Configure OAuth clients if you are building a customer-facing app (see [Authentication](./authentication)).
3. Set up billing and usage monitoring before you deploy production workloads (see [Billing](./billing)).
4. Create routes for any web endpoints you plan to share publicly (see [Domains](./domains)).
