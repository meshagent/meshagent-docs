MeshAgent has two related webhook patterns:

- **Project webhooks** send HTTP events such as room lifecycle and storage events to your endpoint.
- **`room.call` handoffs** are the call flow used when MeshAgent calls a service or toolkit endpoint. In Python, `WebhookServer` handles this flow for you.

## Create a project webhook

You can manage project webhooks from [MeshAgent Studio](../interfaces/meshagent_studio) or with the CLI.

```bash bash
meshagent webhook create \
  --name room-events \
  --url https://example.com/webhook \
  --event room.started \
  --event room.ended \
  --action notify
```

List and delete webhooks:

```bash bash
meshagent webhook list
meshagent webhook delete WEBHOOK_ID
```

Use `--action call` only when you want MeshAgent to send a `room.call` handoff instead of a normal HTTP event.

## Project webhook events

Project webhooks can subscribe to these events:

| Event | Data fields |
| --- | --- |
| `room.started` | `room_name` |
| `room.ended` | `room_name` |
| `storage.file.updated` | `path`, `participant_id` |
| `storage.file.deleted` | `path`, `participant_id` |
| `*` | all supported events |

Standard webhook delivery is JSON:

```json
{
  "event": "room.started",
  "data": {
    "room_name": "my-room"
  }
}
```

## Signatures

If a webhook secret is configured, MeshAgent signs each request and sends a `Meshagent-Signature` header:

- The header value is `Bearer <jwt>`.
- The JWT contains a SHA-256 hash of the payload.
- In Python, `WebhookServer` verifies that signature when you pass `webhook_secret=` or set `MESHAGENT_WEBHOOK_SECRET`.

For local development, you can disable verification with `validate_webhook_secret=False`.

## Handle project webhooks

For standard project webhooks, the normal pattern is a plain HTTP endpoint that accepts JSON and handles the event types you subscribe to.

## Use `WebhookServer` for `room.call`

Use `WebhookServer` when your endpoint needs:

- built-in signature verification
- a default `GET /` health check
- a default `POST /webhook` handler
- support for `room.call` handoffs, including websocket upgrades

Set the webhook secret in your environment if you want the server to verify signed requests:

```bash bash
export MESHAGENT_WEBHOOK_SECRET=your-secret
```

The built-in server listens on port `8080` by default and exposes:

- `GET /` for health checks
- `POST /webhook` for webhook delivery
- `GET /webhook` for websocket upgrades used by `room.call`

For `room.call`, the payload includes:

- `room_name`
- `room_url`
- `token`
- optional `arguments`

That is the same call handoff used by MeshAgent services and toolkit endpoints.
