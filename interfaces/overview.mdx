MeshAgent provides two primary interfaces for humans: [MeshAgent Studio](./meshagent_studio) for building, testing, and operating, and [Powerboards](./powerboards) for delivering user-facing experiences. Both are backed by rooms, so the same agents, tools, and data are available across interfaces. Both are included with a MeshAgent account.

## When to use each interface

**MeshAgent Studio**
- Build and debug agents, tools, and services.
- Inspect live sessions with logs, traces, and metrics.
- Manage projects, rooms, members, secrets, and webhooks.

**Powerboards**
- Share agents with non-technical users.
- Distribute one-click installs for agents and services.
- Provide a clean, focused UI for chat, voice, meetings, and files.

## How they work together

1. Build a service in Studio or locally.
2. Package it as a Service or ServiceTemplate (see [Packaging and Deploying Services](../services_room_containers/packaging_and_deploying/packaging)).
3. Share it in Powerboards as an installable experience or deploy it directly (see [Share agents in Powerboards](../services_room_containers/share_agents_in_powerboards)).
